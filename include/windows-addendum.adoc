=== Suricata

==== Install

Download and install `Npcap` from https://npcap.com/[Npcap website^] 

[CAUTION]
.CAUTION
====
Check `WinPcap API-compatible mode` during install +
Suricata needs `Npcap` to capture packets. It will not start without `WinPcap` mode
====

Download the `.msi` installer from https://suricata.io/download[Suricata Download^] 

[source,powershell]
----
> .\suricata-[version].msi
----

Follow the installer menu:Next[Next] 

===== Interface

[source,powershell]
----
> ipconfig /all
----

.NOTE
NOTE: Take note of the full name of the network adapter, this will be the interface that Suricata monitors

Edit `{suricata-win}\suricata.yaml`
[source,yaml]
----
vars:
  address-groups:
    HOME_NET:"[IP/CIDR]" <.>
----
<.> Change this to match the network

Set the interface under the `pcap` section

[source,yaml]
----
pcap:
  - interface: [ifname] <.>
----
<.> Use the interface name from `ipconfig /all`

.NOTE
NOTE: Windows uses `pcap`, not `af-packet`

===== Rules

.Automatic Rules Download (Administrator CMD)
[source,powershell,subs="attributes+"]
----
> cd "{suricata-win}"
> suricata-update
----

To download rules manually go to https://rules.emergingthreats.net/open/suricata-+[version-num]+/emerging.rules.tar.gz

Extract and copy the `.rules` files into `{suricata-win}\rules\`

===== Usage

.NOTE
NOTE: All commands should be run in an Administrator CMD prompt

.Test config for errors without starting
[source,powershell,subs="attributes+"]
----
> cd "{suricata-win}"
> suricata.exe -T -c suricata.yaml
----

.Start Suricata on interface
[source,powershell,subs="attributes+"]
----
> cd "{suricata-win}"
> suricata.exe -c suricata.yaml -i "[ifname]" <.>
----
<.> Enter your interface name here

.WARNING
WARNING: Do not close this window! Suricata runs in the foreground

.Running as a service (if supported)
[source,powershell,subs="attributes+"]
----
> sc create Suricata binPath=""{suricata-win}\suricata.exe" -c "{suricata-win}\suricata.yaml" -i "[ifname]" --service" start=auto
> net start Suricata
----

.NOTE
NOTE: Not all versions support `--service`

===== Log Files

.Quick one-line alert log, easiest to read
`{suricata-win}\log\fast.log`

.Full JSON log with all detail (`alerts`, `dns`, `http`, `tls`, `flow`)
`{suricata-win}\log\eve.json`

.Performance and packet stats
`{suricata-win}\log\stats.log`

.Suricata's own startup and error log. Check here first if something is wrong
`{suricata-win}\log\suricata.log`

===== Reading Logs (CMD)

.Print All Alerts
[source,powershell,subs="attributes+"]
----
> type "{suricata-win}\log\fast.log"
----

.Search for Malware Signature Hits
[source,powershell,subs="attributes+"]
----
> type "{suricata-win}\log\fast.log" | find "ET MALWARE"
----

.Search for Trojan Signature Hits
[source,powershell,subs="attributes+"]
----
> type "{suricata-win}\log\fast.log" | find "ET TROJAN"
----

.Search for Scan/Recon Activity
[source,powershell,subs="attributes+"]
----
> type "{suricata-win}\log\fast.log" | find "ET SCAN"
----

.Search for Specific IP in Alerts
[source,powershell,subs="attributes+"]
----
> type "{suricata-win}\log\fast.log" | find "ET [IP]"
----

===== Reading Logs (Powershell)

.Show Last 20 Alerts
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\fast.log" -Tail 20
----

.Watch Alerts in Real-Time (like `tail -f` on Linux)
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\fast.log" -Wait
----

.Search for Malware Hits
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\fast.log" | Select-String "MALWARE"
----

.Watch Alerts from `eve.json` in Real-Time
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\eve.json" -Wait | ConvertFrom-Json | Where-Object { $_.event_type -eq "alert" }
----

.Top 20 Most Queried DNS Domains. Good for Finding Beacons
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\eve.json" | ConvertFrom-Json | Where-Object { $_.event_type -eq "dns" } | Select-Object -ExpandProperty dns | Group-Object rrname | Sort-Object Count -Descending | Select-Object -First 20
----

.Clean Alert Summary with IPs, Ports, and Signature Name
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\eve.json" | ConvertFrom-Json | Where-Object { _.event_type -eq "alert" } | Select-Object timestamp, src_ip, dest_ip, dest_port, @{N='signature';E={_.alert.signature}}
----

.This is Something
[source,powershell,subs="attributes+"]
----
> Get-Content "{suricata-win}\log\eve.json" | ConvertFrom-Json | Where-Object { _.event_type -eq "tls" } | Select-Object src_ip, dest_ip, @{N='sni';E={_.tls.sni}}
----

=== Sysmom Script

This script checks all the network connections that sysmon logs and collects only SSH connections. It then stores those in a file named sysmon-ssh-logs.txt. To adjust it, change the port number to any port you want to, and it can show any logs it has on that port. See part 2 to understand how to log more than only one port at a time.

==== Part 1

[source,powershell]
----
# Output file path
$OutputFile = "C:\sysmon-ssh-logs.txt"

# Get Sysmon Event ID 3 events
$Events = Get-WinEvent -FilterHashtable @{
    LogName = "Microsoft-Windows-Sysmon/Operational"
        Id      = 3
} -ErrorAction SilentlyContinue

# Filter for SSH (Destination Port 22)
$SSHEvents = $Events | Where-Object {
    $_.Message -match "DestinationPort:\s+22"
}

# Write results
$SSHEvents | Select-Object TimeCreated, Id, Message |
Format-List | Out-File -FilePath $OutputFile -Encoding UTF8

Write-Output "SSH connection logs written to $OutputFile"
----

==== Part 2

Doing more uses an “or” flag to detect two options. This can be done both with purely ports or any part of the message of the log. The following script checks for both ssh and udp ports:
The script after it also checks for ssh and udp, but does so differently:

[source,powershell]
----
# Output file path
$OutputFile = "C:\sysmon-network-logs.txt"

# Get Sysmon Event ID 3 events
$Events = Get-WinEvent -FilterHashtable @{
    LogName = "Microsoft-Windows-Sysmon/Operational"
        Id      = 3
} -ErrorAction SilentlyContinue

# Filter for SSH (Port 22) OR UDP protocol
$FilteredEvents = $Events | Where-Object {
    $_.Message -match "DestinationPort:\s+22" -or
        $_.Message -match "DestinationPort:\s+53"
}

# Write results
$FilteredEvents | Select-Object TimeCreated, Id, Message |
Format-List | Out-File -FilePath $OutputFile -Encoding UTF8

Write-Output "SSH and UDP connection logs written to $OutputFile"
----

[source,powershell]
----
# Output file path
$OutputFile = "C:\sysmon-network-logs.txt"

# Get Sysmon Event ID 3 events
$Events = Get-WinEvent -FilterHashtable @{
    LogName = "Microsoft-Windows-Sysmon/Operational"
        Id      = 3
} -ErrorAction SilentlyContinue

# Filter for SSH (Port 22) OR UDP protocol
$FilteredEvents = $Events | Where-Object {
    $_.Message -match "DestinationPort:\s+22" -or
        $_.Message -match "Protocol:\s+udp"
}

# Write results
$FilteredEvents | Select-Object TimeCreated, Id, Message |
Format-List | Out-File -FilePath $OutputFile -Encoding UTF8

Write-Output "SSH and UDP connection logs written to $OutputFile"
----

=== Windows Update Registry Repair

[source,powershell]
----
> sc.exe create wuauserv binPath= "C:\Windows\System32\svchost.exe -k netsvcs -p"
> sc.exe config wuauserv type= share
> sc.exe config wuauserv start= delayed-auto
> sc.exe config wuauserv DisplayName= "Windows Update"

> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv" /v ImagePath /t REG_EXPAND_SZ /d "%systemroot%\system32\svchost.exe -k netsvcs -p" /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv" /v Start /t REG_DWORD /d 3 /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv" /v Type /t REG_DWORD /d 32 /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv" /v ObjectName /t REG_SZ /d "LocalSystem" /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv" /v DisplayName /t REG_SZ /d "Windows Update" /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv\Parameters" /v ServiceDll /t REG_EXPAND_SZ /d "%systemroot%\system32\wuaueng.dll" /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv\Parameters" /v ServiceDllUnloadOnStop /t REG_DWORD /d 1 /f
> reg add "HKLM\OFFLINE\ControlSet001\Services\wuauserv\Parameters" /v ServiceMain /t REG_SZ /d "WUServiceMain" /f
----
